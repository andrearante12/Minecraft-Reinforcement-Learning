# logs/

Training output from `malmo/rl/training/train.py`. Each run produces up to three CSV files sharing a common timestamped prefix.

---

## Naming Convention

```
{env}_{algo}_{YYYYMMDD}_{HHMMSS}_{type}.csv
```

Examples:
```
simple_jump_ppo_20260406_181527_episodes.csv
simple_jump_ppo_20260406_181527_updates.csv
simple_jump_ppo_20260406_181527_trajectories.csv
```

Files prefixed with `sample_` are synthetic test data generated by `_gen_sample_logs.py` and are not real training runs.

Curriculum runs use `curriculum_{algo}_{timestamp}` as the prefix instead of `{env}_{algo}`.

---

## File Types

### `_episodes.csv`
One row per episode. Always present.

| Column | Description |
|---|---|
| `episode` | Episode number |
| `reward` | Total undiscounted reward for the episode |
| `steps` | Number of steps taken |
| `outcome` | `landed`, `fell`, `timeout`, or `near_miss` |
| `env` | Environment name (useful for curriculum runs) |
| `timestamp` | Wall-clock time the episode ended (`HH:MM:SS`) |

### `_updates.csv`
One row per policy update. Always present for PPO/DQN runs.

| Column | Description |
|---|---|
| `update` | Update number |
| *(dynamic)* | Loss columns vary by algorithm — PPO: `policy_loss`, `value_loss`, `entropy`, `kl_div`; DQN: `loss` |

### `_trajectories.csv`
One row per environment step. Present only when trajectory logging is active (requires the updated logger with `init_trajectory` / `log_step`). This is the primary file consumed by the visualization tool.

Starts with comment-header lines:
```
# env: simple_jump
# spawn: 0.5,46.0,3.5
# goal: 0.5,45.0,6.5
```

Then CSV columns:

| Column | Description |
|---|---|
| `episode` | Episode number |
| `step` | 0-based step index within the episode |
| `x`, `y`, `z` | Agent world position |
| `yaw` | Agent heading in degrees |
| `pitch` | Agent pitch in degrees |
| `on_ground` | `1` if grounded, `0` if airborne |
| `action` | Action name string (e.g. `sprint_jump`) |
| `reward` | Per-step reward |
| `done` | `1` on the terminal step, `0` otherwise |
| `outcome` | `alive` on non-terminal steps; final outcome on the terminal step |
| `env` | Environment name (tracks env switches in curriculum runs) |

---

## Note for Claude

**Do not read these CSV files speculatively.** They can contain thousands of rows and will consume context without benefit unless the task specifically requires inspecting training results (e.g. debugging an unexpected outcome pattern, verifying log format, or analysing reward trends). If a task involves the visualizer or logger code, read the source files in `visualization/` and `utils/logger.py` instead — the log format is fully described above.

The `_gen_sample_logs.py` script in this directory generates all three file types with synthetic data and is the authoritative reference for the expected format.
